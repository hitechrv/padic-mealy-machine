How it works:

This Mealy machine processes p-adic numbers digit by digit from right to left (least significant first). Each input digit triggers a state transition and produces an output digit, while maintaining internal state for carries or intermediate calculations.

P-adic numbers are written "backwards" - the rightmost digit represents the units place, the next digit represents multiples of p, then p², and so on. The beauty is that every rational number has a unique p-adic representation (possibly infinite to the left).

Important: When you see a final carry > 0, that represents additional digits extending infinitely to the left. For example, ...012₃ with carry=1 actually means ...1012₃.

Try this: Set p=3, choose addition, and enter digits 1, 2, 1 (right-to-left) to see how ...121₃ = 16₁₀ gets doubled to ...1012₃ = 32₁₀. Notice how the final carry=1 represents the leftmost digit!
